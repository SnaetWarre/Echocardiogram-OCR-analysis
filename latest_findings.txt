What Changed To Reach 96.53%
1. Line segmentation improvements
- Replaced coarse stripe placement with gap-aware line placement based on real text pixels.
- Added rescue logic for weak short rows such as `HR`, so low-ink lines do not get merged into the next row.
- Result: recovered missing short lines and removed the main segmentation miss that was still visible in dense panels.

2. GUI overlay improvements
- Overlay boxes are now drawn with crisp pixel-aligned strokes.
- This is mostly a visualization improvement, but it makes reviewing segmentation much easier.

3. ROI detector improvement
- The top-left colored measurement-panel detector no longer rejects tall panels.
- This fixed the worst catastrophic miss:
  `98667422_0054.dcm` went from 0/12 predicted to 12/12 predicted and 10/12 exact.

4. Lexicon / reranker improvements
- The pipeline now rebuilds and uses its lexicon from `labels/labels.json` when that file is newer than the cached artifact.
- Before this, the reranker could use stale label statistics.
- Added repair logic for:
  - unexpected prefixes on families that normally do not use a prefix
  - hallucinated leading digit values such as `12.9` when the lexicon strongly suggests `2.9`

5. OCR text normalization improvements
- Added targeted repairs for repeated real-world OCR mistakes seen in eval:
  - `AVS` -> `AVA`
  - `PRand PG` -> `PRend PG`
  - `LAAS` -> `LAAs`

Quick Pipeline Overview
1. ROI detection
- The pipeline finds the dark blue-gray measurement panel in the top-left of the frame.

2. ROI segmentation
- The ROI is split into separate line boxes.
- The segmenter uses actual text-pixel density and connected components, not just fixed line heights.

3. Line OCR
- Each segmented line crop is sent separately through the OCR engine.
- The pipeline can also try alternative preprocessing views of the same crop.

4. Candidate selection / reranking
- OCR outputs are normalized and scored.
- A lexicon-based reranker uses expected label families, order patterns, units, and numeric ranges to prefer better candidates.

5. Final line comparison
- Predicted lines are compared against the labeled reference lines and scored for exact match, label match, value match, unit match, and prefix match.

Why The Current System Works Better Now
- The line boxes are more accurate, so each OCR crop contains the intended line and less neighboring noise.
- Weak short lines are no longer dropped as easily.
- Tall measurement panels are no longer missed by ROI detection.
- The reranker now uses the current label set and can repair more realistic OCR errors.

Main remaining failure patterns
1. Small numeric OCR mistakes
- Example:
  - expected: `1 LA Diam 4.6 cm`
  - predicted: `1 LA Diam 4.0 cm`
- Example:
  - expected: `1 E' Lat 0.09 m/s`
  - predicted: `1 E' Lat 0.08 m/s`
- Example:
  - expected: `LVESV MOD BP 76.70 ml`
  - predicted: `LVESV MOD BP 76.40 ml`

2. Closely related measurement confusion
- Example family:
  - `MV Dec Slope`
  - `MV E/A Ratio`
- These lines can still be confused in a few dense panels.

3. Label alias disagreement
- Example:
  - expected: `LVEF Biplane 43 %`
  - predicted: `EF Biplane 43 %`
- This is not a detection or segmentation problem anymore; it is mainly an interpretation / canonicalization issue.

4. Prefix noise on a small number of lines
- Example:
  - expected: `2 P Vein D 0.4 m/s`
  - predicted: `$ ^ 2 $ P Vein D 0.4 m/s`
